Skill 入门指南

什么是 Skill？

Skill 是 AI Agent 的「技能包」——一段可复用的知识和指令，让 AI 知道怎么做某件具体的事。不需要写代码，不需要懂编程，人人都可以创建。

🧠

AI 是大脑

大语言模型（LLM）拥有通用的理解和推理能力，但不知道你的具体工作流程。

🔌

MCP 是接口

MCP（模型上下文协议）让 AI 可以连接外部工具和数据，比如读文件、调 API、查数据库。

⚡

Skill 是肌肉记忆

Skill 告诉 AI 「在什么情况下、用什么工具、按什么步骤」完成一件具体的事。

Skill 如何运作？

当你安装了一个 Skill，AI Agent 会在合适的时候自动调用它，就像一个有经验的同事。

1

你发出请求

「帮我把这个设计文档生成测试用例」

2

Agent 识别 Skill

Agent 在已安装的 Skills 中匹配到「QA Test Generator」

3

按 Skill 指引执行

Agent 按照 Skill 定义的步骤和格式，调用相关工具完成任务

4

交付结果

输出结构化的测试用例文档，直接可用

一个 Skill 长什么样？

Skill 的核心就是一个 SKILL.md 文件——纯文本的 Markdown，完全透明可读。

---
name: game-asset-generator
description: 批量生成游戏美术资产
version: 2.1.0
---

# Game Asset Generator

## 什么时候使用
当用户需要批量生成角色贴图、道具
图标、场景概念图时，使用此技能。

## 执行步骤
1. 确认美术风格（卡通/写实/像素）
2. 收集参考图和风格描述
3. 调用 Stable Diffusion API 生成
4. 按照项目命名规范输出文件
5. 生成资产预览报告

## 输出格式
- PNG, 2048x2048, sRGB 色彩空间
- 命名: {项目代号}_{资产类型}_{编号}

📋 元数据（Frontmatter）

Skill 的名字、描述、版本号。Agent 用这些信息判断什么时候该调用这个 Skill。

🎯 触发条件

明确告诉 Agent 在什么场景下使用这个 Skill，避免误触发。

📝 执行步骤

一步步的操作指引——就像你写给新同事的工作流程 SOP。这是 Skill 的核心价值。

📁 输出规范

定义交付物的格式、命名和质量标准，确保结果一致、可用。

🔧 附加脚本（可选）

复杂 Skill 可以附带脚本文件，由 Agent 在执行时调用。不是必须的，纯文本 Skill 同样好用。

Skill 能帮你做什么？

不同岗位、不同场景，Skill 都能将你的经验转化为可复用的 AI 能力。

策划

自动生成支线任务和 NPC 对话

将世界观设定和角色关系注入 Skill，AI 可以批量生成符合设定的支线任务文案和 NPC 对话脚本。

开发

代码审查与引擎模板生成

让 AI 按照团队的编码规范审查提交代码，或根据需求自动生成 UE5/Unity 的 Component 模板。

技术美术

批量生成和处理美术资产

定义风格、分辨率和命名规范，AI 调用绘图工具批量产出贴图、概念图、UI 素材。

QA

从设计文档自动生成测试用例

读取 GDD 文档，自动识别功能点和边界条件，输出兼容 TAPD/Jira 的结构化测试用例。

游戏运营

玩家行为分析与自动报告

用自然语言查询数据仓库，AI 按照 Skill 定义的分析框架生成留存、付费、活跃等运营报告。

商业化

广告素材自动化产出

根据投放策略自动生成多尺寸、多风格的广告素材和文案变体，加速 A/B 测试覆盖。

如何创建你自己的 Skill？

你不需要是程序员。任何人都可以把自己的工作经验转化为 Skill——用自然语言写就行。LLM 可以帮你完成从想法到代码的全过程。

🤖

方式一：用 AI 帮你写 推荐

用 WorkBuddy、CodeBuddy、Claude、ChatGPT 等任何 LLM，把你的经验描述出来，让 AI 帮你生成 SKILL.md。

  1. 描述你的工作流程 — 「我平时做 XX 的时候，第一步是...第二步是...」
  2. 让 AI 生成 Skill 文件 — 「请帮我把这个流程写成一个 SKILL.md」
  3. 测试和迭代 — 安装到你的 Agent 里试用，不好就改
  4. 分享 — 发布到 AI World / SkillHub，让团队都能用
✍️

方式二：自己动手写

如果你对 Markdown 格式熟悉，可以直接创建 SKILL.md 文件。写好后放到你的 Agent 技能目录即可。

  1. 新建文件夹 — 在 ~/.skills/ 下创建以技能命名的文件夹
  2. 写 SKILL.md — 包含元数据、触发条件、执行步骤、输出规范
  3. （可选）附加脚本 — 如有需要，放入 Python/Shell 脚本
  4. 测试 — 在 Agent 中触发场景，验证效果

💡 写 Skill 的最佳实践

从你最熟悉的重复性工作开始。如果一件事你每周都在做、每次流程差不多，那它就是一个完美的 Skill 候选。 写 Skill 就像写给新同事的工作交接文档——描述清楚「什么时候做」「怎么做」「做到什么程度算合格」就够了。

推荐工具：腾讯 WorkBuddy（兼容 OpenClaw 技能生态，支持 Skill 导入和创建）、 CodeBuddy（AI 编程助手，适合写复杂 Skill）、 Claude、ChatGPT 等通用 LLM 同样可以帮你生成 Skill。

📝 鼓励投稿：把你的经验变成文章和 Skill

我们鼓励每一位同事将自己的 AI 实践经验写成知识库文章。好的文章可以进一步转化为 Skill，发布到 AI World 技能商店供全团队复用。 一篇好的经验文章 + 一个可安装的 Skill = 最高效的知识传递方式。

文章和 Skill 不需要完美，能用就发。迭代才是正道。
